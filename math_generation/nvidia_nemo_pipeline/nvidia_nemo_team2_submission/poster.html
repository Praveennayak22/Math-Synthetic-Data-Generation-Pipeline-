Abstract

Core Innovation: Training state-of-the-art LLMs requires large-scale, high-quality datasets. Rather than expensive manual curation, we automate synthesis of verified reasoning datasets. Our pipeline generates diverse, difficulty-scaled problems across two domains: code and mathematics.

Code Generation Module: We implement a KodCode-style dual-model architecture where DeepSeek-V3.2 generates problem variants with 8 mutation types and 20 domain injections, while GPT-OSS-120B independently solves them (no reference access, ensuring anti-circular testing). Execution-verified via pytest — yielding 3,008 verified entries (1,456 SFT + 3,008 RL, 42.3% pass rate in healthy difficulty range).

Math Generation Module: We synthesize 53,288 verified math problems via: (1) aggregate 10,506 seeds from 8 HF datasets, (2) apply 5 Evol-Instruct mutations via Kimi-K2.5 → 5.1× expansion, (3) standardize MCQ→open-ended format, (4) generate reasoning traces with symbolic verification (96% pass), (5) tag complexity levels 0–4 for SFT/RL routing.

Quality Validation Loop: The NeMo Pipeline has two synergistic branches:

Evaluation Branch (Steps 1–4): Benchmarks 5 SOTA models (GLM-5-FP8, Minimax-M2.5, Kimi-K2.5, DeepSeek-V3.2, GPT-OSS-120B) across 4 HuggingFace datasets (GAIR/LIMO, simplescaling/s1K, TIGER-Lab/TheoremQA, twinkle-ai/tw-math-reasoning-2k) to identify which model generates the highest-quality problems. Result: Kimi-K2.5 identified as best performer.

Generation Branch (Steps 5–7): Uses the best model identified (Kimi-K2.5) to synthetically generate new problems by leveraging HF datasets as inspiration. Step 5 generates 400+ original math problems (8 topics × 5 difficulty levels). Step 6 creates dataset-inspired variants with progressive difficulty scaling and duplicate deduplication. Step 7 schema-converts external datasets (DeepMath-103K, AutoMathText, AM-DeepSeek-R1) into unified format for downstream training. Output: 53,288 verified samples with zero repetition, difficulty-tagged 0–4.

Introduction

Problem: Synthetic data addresses training data scarcity, but generation lacks rigor:

  • Scale: Manual dataset curation bottleneck; need automated 10K→53K+ expansion
  • Quality: Without symbolic math verification, noisy labels corrupt model training
  • Diversity: Single-model generation creates distribution collapse; need Evol-Instruct + multi-model variety
  • Safety: Code generation risks malicious injection; need 19-pattern safety scanner + execution verification
  • Anti-circular: If same model generates tests AND solutions, no true diversity or solvability guarantee
  • Validation: How to know generated data actually improves LLM training? Need evaluation feedback loop

Our Solution: Three synergistic generation pipelines: (1) Code: DeepSeek generates → GPT-OSS independently solves → pytest verifies (anti-circular by design). (2) Math: Evol-Instruct via Kimi-K2.5 → 5 mutations each seed → SymPy symbolic verification. (3) Validation: 5-model benchmarking validates generated quality, closes data quality loop.

🔒 Code Safety Scanner — 19-Pattern Filter

All generated code passes through a two-tier static safety scanner before entering the dataset. Blocks dangerous system calls and prevents data leakage.

🚫 Tier 1 — Hard Reject
os.system subprocess eval( exec( __import__ open( shutil socket ctypes pickle marshal
→ Entry discarded entirely
⚠️ Tier 2 — Warn & Flag
requests urllib http.client ftplib paramiko cryptography hashlib base64
→ Flagged, human review queue
Result: 4,099 total rejections in Batch 1 — 0 unsafe entries in final dataset

🚀 Generation Achievements

Code Generation: DeepSeek-V3.2 synthesized diverse problem variants; GPT-OSS-120B independently solved without reference access. Anti-circular design ensures 42.3% pass rate (healthy 35–50% difficulty range, ideal for RL curriculum learning).

Math Generation: Kimi-K2.5 applied 5 Evol-Instruct mutations to 10,506 seeds, achieving 5.1× expansion (53,288 final samples) with 96% verification rate. No manual problem writing required — fully automated synthesis.

Diversity by Design: Code uses 8 mutation types × 20 domain injections. Math uses 5 Evol-Instruct strategies. Generated problems span difficulty levels 0–4 (7.5% → 8.0% coverage), enabling nuanced SFT/RL routing.

Schema Unification: External datasets converted to common standard, enabling downstream fusion with generated data.

Validation Loop (Quality Assurance)

Generated data must be validated before training. Our NeMo Evaluation Branch benchmarks 53,288 samples via 5 models × 4 datasets:

  • Kimi-K2.5 achieves highest accuracy on LIMO, s1K, TheoremQA → validates Kimi generation quality
  • Multi-model agreement indicates diversity of generated problems (prevents distribution collapse)
  • 7-tier answer extraction verifies reasoning completeness
